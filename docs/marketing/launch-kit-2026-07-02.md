# WoopKid World Launch Kit

Drafts in Ron's voice, ready to fire per the 30-day playbook. Nothing here is published; every post is Ron's manual action from his own accounts. Facts used: 15-game roster, ad-free, no tracking, no accounts, single-file vanilla JS + Canvas, localStorage only, open source on GitHub.

One personal note before using: several drafts lean on the "I built this for my kids" angle the research says is the strongest hook. Confirm you want that framing before posting; strip it where it does not feel right.

---

## 1. Show HN (one shot, fire only when the site is polished)

**Title:**
Show HN: I built 15 ad-free browser games for kids. No accounts, no tracking

**First comment (post immediately after submitting):**

Every "free games for kids" site is an ad minefield. Autoplay video ads next to games aimed at 6-year-olds, fake download buttons, trackers everywhere. I got tired of it and built the alternative.

WoopKid World is 15 browser games for kids ages 5-12. The rules I set for myself:

- No ads, ever. No ad networks anywhere near it.
- No accounts, no tracking, no cookies, no data collection. Progress lives in localStorage.
- Every game is a single self-contained HTML file. Vanilla JS and Canvas, no frameworks, no build step. View source and the whole game is right there.
- Mobile-first, 60fps on a school Chromebook.

Start with [GAME NAME], it is the one adults keep replaying.

The whole thing is open source: https://github.com/Ronkimhi/woopkid-games

Happy to answer anything about the tech or the design decisions. The hardest part was not the code. It was figuring out what makes a kid hand the tablet to a friend and say "you try it."

**Rules reminders:** post Tue-Thu 8-9am ET, stay in the comments all day, never ask anyone to upvote.

## 9. Boilerplate (100 words, for quote pipelines and press)

WoopKid World (woopkid.com) is a free, ad-free game site for kids ages 5-12. It hosts 15 original browser games that run on any device with zero ads, zero tracking, zero accounts and zero purchases. All progress stays on the player's device. Every game is a single open-source HTML file built with vanilla JavaScript, so parents, teachers and school IT teams can verify every claim. WoopKid was built by Ron Kimhi, a tech executive and parent, as a protest against "free kids games" sites that monetize children's attention with ad networks.
